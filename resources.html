Resources & API

Local server commands, REST endpoints for sensor fusion, and health check—useful when integrating a Pi or custom firmware.

Run the dev server

From the project folder:

cd "/path/to/News 3"
node server.js

Then open http://localhost:3000 — static pages and API share the same origin.

Sensor API (Express)

Method Path Purpose
GET /api/status JSON: movement, rfSignal, heartRate, gaze, objects, aiAlert
POST /api/update-sensors Merge JSON body into live state (body: application/json)
GET /health {"ok":true} liveness probe

Arduino → browser → JSON API

Arduino over USB sends text lines. The Live Lab can treat a line as JSON and, if you enable POST JSON to API, forward that object to POST /api/update-sensors. Same-origin rules apply: open the site from http://localhost:3000 while server.js runs. See New user guide → JSON section.

Example: POST sensor update

curl -X POST http://localhost:3000/api/update-sensors \
  -H "Content-Type: application/json" \
  -d '{"movement":"Motion","heartRate":78,"aiAlert":"Low Risk"}'

Deploying static pages

For GitHub Pages, publish index.html, site.css, live-lab.html, app.js, and related HTML files. Web Serial still requires a supported browser; the API must be hosted separately or replaced with your backend.
