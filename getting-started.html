New user guide

Follow these steps in order the first time you use OmniGuard. If something fails, check the troubleshooting section at the bottom.

First-time setup

  1. Use a supported browser. Web Serial works in Chrome or Edge (Chromium). Safari and Firefox do not support Web Serial for this workflow.
  2. Serve the site over HTTP or HTTPS. Open the project from a local server (for example Node: node server.js in the project folder) and visit http://localhost:3000. Web Serial needs a secure or localhost context.
  3. Start the API (optional but recommended). The same server.js exposes GET /api/status so the Live Lab sensor tiles update. Without it, tiles show “API: offline” but Web Serial still works.
  4. Open Live Lab. Go to Live Lab, choose baud rate (often 115200), click Connect, and pick your Arduino/USB serial device.
  5. Send numeric lines. Paste the minimal sketch below into Arduino IDE, upload, then open Live Lab.

Minimal test sketch (Arduino IDE)

Copy everything in the box into a new sketch, upload to your board, close Serial Monitor in Arduino IDE, open Live Lab, set baud to 115200, click Connect. You should see numbers in the stream and a moving chart (A0 = analog pin 0). 

void setup() {
  Serial.begin(115200);
}

void loop() {
  Serial.println(analogRead(A0));
  delay(100);
}

USB-C (Type-C) and Arduino

The website uses the same Web Serial API no matter whether you plug in with USB-A or USB-C. If connection fails with Type-C, the problem is almost always cable, port, or driver—not the page code.

  • Charge-only USB-C cables — Very common. They only carry power, so the computer never sees a serial device. Use a cable that says data / USB 2.0 or 3.x, or try a different cable you know works for file transfer or another device.
  • Dongles and hubs — Cheap USB-C hubs sometimes drop serial or enumerate oddly. Try plugging the board directly into the laptop’s USB-C port, or use a simple USB-C→USB-A adapter plus a known data USB-A cable to the Arduino.
  • Drivers — Boards with CH340, CP2102, or FTDI need the right driver on macOS/Windows. Install the driver from the chip maker or your board vendor, then unplug and replug the board.
  • Port in use — Close Arduino IDE Serial Monitor, other terminal apps, and any second browser tab that might have the port open. Only one program can use the port at a time.
  • Pick the right device — In the browser’s port list, choose the entry that matches your board (often “usbmodem…”, “tty.usbserial…”, “COM…”, or “Arduino”).

Sending JSON from Arduino (USB-C or USB-A)

The board only speaks serial over USB—the cable shape (Type-C vs A) does not matter. Your website reads bytes with Web Serial. To use JSON, print one JSON object per line (then println so each line ends). The Live Lab will JSON.parse lines that start with {.

  • Chart — The graph uses the first numeric field it finds, preferring keys like value, telemetry, heartRate, bpm, distance.
  • Upload to your server — In Live Lab, enable POST JSON to API. Each parsed line is sent as the body of POST /api/update-sensors (throttled ~every 400 ms). Run node server.js on the same machine so the browser can reach the API.
  • Field names — Use the same names your tiles expect: movement, rfSignal, heartRate, gaze, objects, aiAlert (see Resources).

Minimal Arduino example (escape quotes carefully):

void setup() {
  Serial.begin(115200);
}
void loop() {
  Serial.println("{\"heartRate\":72,\"movement\":\"Stable\",\"rfSignal\":\"Clear\"}");
  delay(500);
}

For many fields, the ArduinoJson library builds valid JSON without manual escaping.

Understanding the dashboard

  • Left panel — Web Serial stream, Chart.js plot, connect/disconnect.
  • Tiles — Ultrasonic, RF, heart rate, gaze, objects, and AI alert come from /api/status when the server is running.
  • Click a sensor tile — Opens a short explanation modal (architecture and product modals work the same way).

Troubleshooting

  • Type-C not working — See USB-C section above: data cable, direct port, drivers.
  • No serial port in the list — Replug USB, try another cable, install CH340/CP210x/FTDI driver, check System Information / Device Manager for the device.
  • Connect fails or “Failed to open serial port” — Close Arduino IDE Serial Monitor and any other serial app; try another USB port or cable.
  • Connected but no lines in the stream / flat chart — Use Serial.println(...) so each message ends with a newline (Serial.print alone may buffer until a line ends). Match baud: Live Lab dropdown must equal Serial.begin(9600) or 115200, etc. The chart only moves if the line contains a number or JSON with a numeric field (see JSON section).
  • API offline — Run node server.js from the project directory; see Resources for endpoints.
